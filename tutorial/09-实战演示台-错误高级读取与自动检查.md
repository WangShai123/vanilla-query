## 开场口播

前两集实战里，我们已经完成了列表、详情和写操作。

这一集处理更接近真实线上问题的部分：

- 接口失败怎么办。

## 1. 错误场景面板

页面里有一个“错误、重试、超时、取消”面板。

它包含四个按钮：

- 运行重试
- 业务错误
- 超时
- 取消

口播：

这四个按钮分别对应生产环境里最常见的异常路径。

它们不是为了写 UI 特效，而是为了确认 query 状态能正确表达请求生命周期。

## 2. 重试接口

模拟接口：

```js
async retryReport() {
  await delay(180);
  counters.retryFailures += 1;

  if (counters.retryFailures < 3) {
    const error = new Error('报表服务临时不可用');
    error.status = 503;
    throw error;
  }

  return { ok: true, attempts: counters.retryFailures };
}
```

口播：

前两次抛 503。

第三次成功。

这样可以直观看到 retry 是否生效。

## 3. 创建重试 query

代码：

```js
const query = createQuery({
  client,
  queryKey: ['report', Date.now()],
  retry: 2,
  retryDelay: 120,
  queryFn: () => api.retryReport(),
});

const result = await query.promise();
```

讲解：

- `retry: 2` 表示失败后再试两次。
- 加上首次请求，总共最多 3 次。
- `retryDelay` 控制每次重试前等待多久。

口播：

这里用 `Date.now()` 放进 queryKey，是为了每次点击都创建新的请求身份，不受上一次缓存影响。

## 4. 业务错误

模拟接口：

```js
async businessError() {
  await delay(160);
  return {
    success: false,
    code: 'NO_ACCESS',
    message: '当前账号没有审批权限',
  };
}
```

调用：

```js
const query = createQuery({
  client,
  queryKey: ['business-error', Date.now()],
  queryFn: () => api.businessError(),
});

try {
  await query.promise();
} catch (error) {
  console.log(error.name, error.code, error.message);
}
```

口播：

默认 normalize 会识别 `success: false`。

然后抛出 `BusinessError`。

业务页面可以根据 `error.code` 做不同提示。

## 5. 超时

代码：

```js
const query = createQuery({
  client,
  queryKey: ['timeout', Date.now()],
  timeout: 120,
  queryFn: (context) => api.slow(context),
});
```

口播：

`api.slow` 会等待 1200 毫秒。

query 的 `timeout` 设置为 120 毫秒。

所以它会抛出 `TimeoutError`。

超时时，内部也会尝试 abort 当前请求。

## 6. 取消

代码：

```js
const query = createQuery({
  client,
  queryKey: ['abort', Date.now()],
  queryFn: (context) => api.slow(context),
});

await delay(80);
query.abort();
```

口播：

取消请求后：

- 当前 `AbortController` 会触发。
- `isFetching` 变成 false。
- 旧请求即使后面返回，也不会写回当前状态。

这适合离开页面、关闭弹窗、取消搜索等场景。

## 7. Suspense 读取

代码：

```js
const query = createQuery({
  client,
  queryKey: ['suspense', Date.now()],
  suspense: true,
  queryFn: async () => {
    await delay(220);
    return 'suspense data';
  },
});

try {
  query();
} catch (promise) {
  await promise;
}
```

口播：

开启 `suspense: true` 后，如果首次请求还没完成，读取 `query()` 会抛出 Promise。

普通业务页面不一定需要这个能力。

但如果你要自己实现 Suspense 风格的 fallback，它就是必要的基础。

## 8. throwErrors

代码：

```js
const query = createQuery({
  client,
  queryKey: ['throw-errors', Date.now()],
  throwErrors: true,
  queryFn: () => api.businessError(),
});
```

读取：

```js
try {
  query();
} catch (error) {
  console.log(error.message);
}
```

口播：

默认情况下，错误保存在 `query.state.error`。

开启 `throwErrors` 后，读取 `query()` 会抛出当前错误。

这个更适合统一错误边界或高级控制流。

普通页面直接读 `query.state.error` 更直观。

## 9. query.subscribe

代码：

```js
const unsubscribe = query.subscribe(() => {
  calls += 1;
});

await query.promise();
unsubscribe();
```

口播：

`query.subscribe` 用来订阅单个 query 的状态变化。

大部分 UI 使用 `createEffect` 就够了。

但如果你要接入外部工具、统计或日志系统，subscribe 会更直接。

## 10. client.subscribe

代码：

```js
client.subscribe((event) => {
  addLog(`client.${event.type}`, {
    key: event.key?.slice?.(0, 80),
  });
});
```

口播：

`client.subscribe` 监听的是整个 query client 的事件。

演示页右侧的事件日志就是基于它实现的。

你可以看到：

- set
- fetch
- success
- error
- invalidate
- remove
- clear

这些事件适合做开发调试，也适合接入 devtools。

## 11. 自动检查 runner

演示页里有一个“运行全部检查”按钮。

核心工具函数：

```js
const assertCheck = async (name, fn) => {
  setCheck(name, 'running');

  try {
    const message = await fn();
    setCheck(name, 'pass', message || '');
  } catch (error) {
    setCheck(name, 'fail', error.message);
  }
};
```

口播：

这个 runner 不是替代单元测试。

它的价值是把业务场景用可视化方式跑一遍。

录课时，观众可以看到每个 API 的实际效果。

开发时，也可以把它当作手动回归测试页。

## 12. 自动检查覆盖项

代码里覆盖了这些检查：

```js
await assertCheck('stableHash 对象 key 稳定', () => {});
await assertCheck('基础 products query 成功', async () => {});
await assertCheck('getQueryData 读取商品缓存', () => {});
await assertCheck('响应式 queryKey 切换保留旧数据', async () => {});
await assertCheck('invalidateQueries 前缀失效', () => {});
```

后面还会触发按钮：

- `prefetchQuery`
- `mutate`
- `retry`
- `BusinessError`
- `timeout`
- `abort`
- `suspense`
- `throwErrors`
- `query.subscribe`

口播：

这个页面不是为了追求自动化测试覆盖率。

它更像是一个交互式 API 操作台。

能看到状态，也能看到事件。

## 13. 如何运行

从项目根目录启动任意静态服务。

例如使用 Vite：

```bash
./node_modules/.bin/vite --host 127.0.0.1 --port 4173
```

打开：

```text
http://127.0.0.1:4173/tests/test.html
```

点击“运行全部检查”。

正常情况下应该看到：

- 通过数增加。
- 失败数为 0。
- 右侧出现 client 事件日志。
- 商品、详情、Todo、错误、高级读取面板都有可见反馈。
