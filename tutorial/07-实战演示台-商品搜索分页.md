口播：

它有搜索词、分类、页码。

## 3. 创建独立 client

代码：

```js
const client = createQueryClient({
  cacheMax: 80,
  cacheTime: 1000 * 60 * 8,
});
```

口播：

演示页使用独立 client。

这样它的缓存、pending 请求和事件日志都和默认 client 隔离。

真实项目里，大多数时候可以直接使用默认 `queryClient`。

在复杂后台、测试页面、微前端或多业务域里，可以创建独立 client。

## 4. 准备响应式参数

代码：

```js
const [search, setSearch] = createSignal('');
const [category, setCategory] = createSignal('all');
const [page, setPage] = createSignal(1);
```

口播：

这三个 signal 是页面上的本地状态。

搜索框改 `search`。

分类下拉框改 `category`。

分页按钮改 `page`。

query 会读取这些 signal，自动建立依赖。

## 5. 模拟商品接口

简化代码：

```js
const api = {
  async products({ queryKey, signal }) {
    await delay(420, signal);

    const [, params] = queryKey;
    const keyword = params.search.trim().toLowerCase();

    const filtered = database.products.filter((product) => {
      const matchKeyword = product.name.toLowerCase().includes(keyword);
      const matchCategory =
        params.category === 'all' || product.category === params.category;

      return matchKeyword && matchCategory;
    });

    return {
      success: true,
      data: {
        items,
        page,
        pageSize,
        total,
        totalPages,
      },
    };
  },
};
```

口播：

这里用本地数组模拟后端。

`delay` 支持 `signal`，所以后面取消请求时可以真的触发 Abort。

返回结构使用 `{ success, data }`，刚好能走默认 normalize。

## 6. 创建商品列表 query

完整核心代码：

```js
const productsQuery = createQuery({
  client,
  queryKey: () => [
    'products',
    {
      search: search(),
      category: category(),
      page: page(),
    },
  ],
  keepPreviousData: true,
  staleTime: 5000,
  cacheTime: 1000 * 60 * 5,
  queryFn: (context) => api.products(context),
});
```

逐句讲解：

- `client`：使用演示页自己的 query client。
- `queryKey`：由搜索词、分类、页码组成。
- `keepPreviousData: true`：参数变化时先保留旧列表。
- `staleTime: 5000`：5 秒内相同 key 直接使用 fresh cache。
- `cacheTime`：缓存保留 5 分钟。
- `queryFn`：读取 queryKey 参数并请求数据。

口播：

这个 query 是实战页的核心。

用户操作表单时，不需要手动调用请求函数。

只要 signal 变化，queryKey 变化，query 就会自动请求。

## 7. 绑定搜索和分页控件

代码：

```js
productSearchInput.addEventListener('input', (event) => {
  setSearch(event.currentTarget.value);
  setPage(1);
});

categorySelect.addEventListener('change', (event) => {
  setCategory(event.currentTarget.value);
  setPage(1);
});

nextPageButton.addEventListener('click', () => {
  setPage((value) => value + 1);
});
```

口播：

搜索和分类变化后，把页码重置为 1。

分页按钮只改 `page`。

这些操作都不直接调用 `productsQuery.refetch()`。

因为参数变化本身就已经足够驱动请求。

## 8. 渲染状态条

代码：

```js
const renderProducts = () => {
  const data = productsQuery();

  productsStateStrip.innerHTML = [
    statePill('status', productsQuery.state.status),
    statePill('fetch', productsQuery.state.fetchStatus),
    statePill('stale', productsQuery.state.isStale ? 'yes' : 'no'),
    statePill('page', data?.page ?? page()),
    statePill('total', data?.total ?? 0),
    statePill('key', productsQuery.key().slice(0, 28)),
  ].join('');
};
```

口播：

这个状态条是为了教学。

业务页面不一定要把这些字段都显示出来。

但录课时可以直观看到状态变化：

- 首次加载时 `status: pending`。
- 请求中 `fetch: fetching`。
- 保留旧数据刷新时 `stale: yes`。
- 请求成功后 `status: success`。

## 9. 渲染列表

代码：

```js
if (productsQuery.state.isLoading) {
  productsBody.innerHTML = `<tr><td colspan="5">正在加载商品...</td></tr>`;
  return;
}

if (productsQuery.state.isError) {
  productsBody.innerHTML = `<tr><td colspan="5">${productsQuery.state.error.message}</td></tr>`;
  return;
}

const rows = data?.items ?? [];
```

口播：

这里不要只看一个 `loading`。

首次进入页面没有数据，显示 loading。

翻页时有旧数据，就继续显示旧列表，同时状态条显示正在刷新。

这就是 `isLoading` 和 `isFetching` 的区别。

## 10. 手动刷新和重新加载

代码：

```js
refreshProductsButton.addEventListener('click', () => {
  productsQuery.refetch({ meta: { source: 'manual-refresh' } });
});

reloadProductsButton.addEventListener('click', () => {
  productsQuery.reload({ meta: { source: 'manual-reload' } });
});
```

口播：

`refetch` 更像后台刷新，默认保留旧数据。

`reload` 更像重新加载，默认不保留旧数据。

同一个接口，交互意图不同，状态体验也不同。

## 11. 预取下一页

代码：

```js
prefetchProductsButton.addEventListener('click', async () => {
  const data = productsQuery();
  const nextPage = Math.min(
    (data?.page ?? page()) + 1,
    data?.totalPages ?? page()
  );

  await client.prefetchQuery({
    queryKey: [
      'products',
      {
        search: search(),
        category: category(),
        page: nextPage,
      },
    ],
    staleTime: 5000,
    queryFn: (context) => api.products(context),
  });
});
```

口播：

预取不会直接改变当前 query 的 data。

它只是提前把下一页写进缓存。

用户再点下一页时，如果缓存还 fresh，就可以快速显示。
