- 商品详情与缓存命中。
- Todo 乐观更新、回滚、同步和失效。

本集会覆盖：

- `enabled`
- `select`
- `getQueryData`

## 开场口播

上一集我们完成了商品列表。

真实后台里，列表后面通常会接详情。

详情页会反复打开，如果每次都请求，体验会慢。

同时，后台也会有写操作，比如新增任务、勾选任务、保存后刷新列表。

这一集就把“详情缓存”和“乐观更新”两个场景补上。

## 1. 详情场景的业务目标

页面上有一个“详情与缓存命中”面板。

它要支持：

- 从列表选择商品。
- 请求商品详情。
- 预取当前商品详情。
- 读取详情缓存。
- 让详情缓存失效。
- 删除详情缓存。

口播：

列表和详情是 queryKey 设计最常见的组合。

列表 key 通常是：

```js
['products', params];
```

详情 key 通常是：

```js
['product', id];
```

这样保存某个商品时，可以只失效对应详情，也可以失效整个商品列表。

## 2. selectedProductId

代码：

```js
const [selectedProductId, setSelectedProductId] = createSignal(1);
```

口播：

当前选中的商品 id 是一个本地状态。

详情 query 读取它作为 queryKey 的一部分。

当选中商品变化时，详情 query 自动切换。

## 3. 创建详情 query

代码：

```js
const productDetailQuery = createQuery({
  client,
  enabled: () => selectedProductId() !== null,
  queryKey: () => ['product', selectedProductId()],
  staleTime: 30000,
  queryFn: (context) => api.productDetail(context),
  select: (product) => ({
    id: product.id,
    name: product.name,
    status: product.status,
    price: product.price,
    stock: product.stock,
    description: product.description,
    fetchedAt: product.fetchedAt,
  }),
});
```

逐句讲解：

- `enabled`：没有选中商品时不请求。
- `queryKey`：详情缓存按商品 id 区分。
- `staleTime: 30000`：30 秒内详情保持 fresh。
- `select`：只保留页面需要的字段。

口播：

`select` 不是渲染函数。

它是在数据进入 state/cache 前做一次数据裁剪或派生。

这适合过滤字段、排序、补充计算字段。

## 4. 选择列表第一项

代码：

```js
selectProductButton.addEventListener('click', () => {
  const first = productsQuery()?.items?.[0];
  if (first) setSelectedProductId(first.id);
});
```

口播：

点击按钮只改变 `selectedProductId`。

详情 query 会自己响应这个变化。

业务代码不需要手动调用详情请求。

## 5. 读取详情缓存

代码：

```js
const cached = client.getQueryData(['product', selectedProductId()]);
```

口播：

`getQueryData` 是同步读取缓存。

它不会发请求。

适合做调试、立即回填、或者判断某个数据是否已经预取。

演示页把缓存内容显示在 `manualCacheOutput` 里。

## 6. 预取详情

代码：

```js
await client.prefetchQuery({
  queryKey: ['product', selectedProductId()],
  staleTime: 30000,
  queryFn: (context) => api.productDetail(context),
  select: (product) => ({
    id: product.id,
    name: product.name,
    status: product.status,
    price: product.price,
    stock: product.stock,
    description: product.description,
    fetchedAt: product.fetchedAt,
  }),
});
```

口播：

预取详情的典型使用场景是：

- 鼠标 hover 到列表行。
- 即将打开详情抽屉。
- 跳转详情页前提前加载。

预取完成后，真正创建或切换到同一个详情 query 时，可以直接命中缓存。

## 7. 失效详情

代码：

```js
const count = client.invalidateQueries(['product', selectedProductId()]);
```

口播：

失效不会删除缓存。

它只是把缓存标记成 stale。

这样 UI 仍然可以先展示旧详情，再后台刷新。

## 8. 删除详情缓存

代码：

```js
const count = client.removeQueries(['product', selectedProductId()]);
productDetailQuery.remove();
```

口播：

删除缓存更强。

如果商品已经不存在，或者你明确不希望继续展示旧详情，就可以 remove。

`productDetailQuery.remove()` 还会重置当前 query 状态。

## 9. Todo 场景的业务目标

Todo 面板要演示写操作：

- `initialData` 先给列表一个初始值。
- `mutate` 做乐观新增。
- `mutate` 切换第一项状态。
- 模拟失败后回滚。
- 保存到模拟服务端后，失效并刷新。

口播：

读操作解决的是“怎么拿数据”。

写操作解决的是“用户操作后 UI 怎么立刻响应，以及服务端同步后怎么保持一致”。

## 10. 创建 Todo query

代码：

```js
const todosQuery = createQuery({
  client,
  queryKey: ['todos'],
  initialData: database.todos.map((todo) => ({ ...todo })),
  staleTime: 20000,
  queryFn: (context) => api.todos(context),
});
```

讲解：

- `initialData` 让列表初始就是数组。
- `staleTime` 让短时间内的重复读取不必重新请求。
- `queryKey` 固定为 `['todos']`。

## 11. 乐观新增

代码：

```js
addTodoButton.addEventListener('click', () => {
  const todo = {
    id: Date.now(),
    text: '乐观任务',
    done: false,
  };

  todosQuery.mutate((todos) => [todo, ...(todos ?? [])]);
});
```

口播：

`mutate` 会立即更新当前 state。

默认也会写入缓存。

所以所有读取 `['todos']` 的地方都能看到新数据。

## 12. 切换 Todo 状态

代码：

```js
todosQuery.mutate((todos) =>
  (todos ?? []).map((todo, index) =>
    index === 0 ? { ...todo, done: !todo.done } : todo
  )
);
```

口播：

这里的重点是不要原地改数组和对象。

返回新数组、新对象，状态变化会更清晰。

## 13. 失败回滚

代码：

```js
const previous = todosQuery();

todosQuery.mutate((todos) => [
  { id: Date.now(), text: '即将回滚的任务', done: false },
  ...(todos ?? []),
]);

await delay(180);
todosQuery.mutate(previous);
```

口播：

乐观更新前先保存旧数据。

如果接口失败，把旧数据写回去。

这就是最基本的回滚策略。

真实项目里，可以在 catch 分支里回滚。

## 14. 服务端同步后失效刷新

代码：

```js
database.todos = todosQuery().map((todo) => ({ ...todo }));
client.invalidateQueries(['todos']);
await todosQuery.refetch();
```

口播：

保存成功后，把当前列表写入模拟服务端。

然后让 `['todos']` 失效。

最后手动 `refetch`，保证页面和服务端重新对齐。
