# 第 4 集：搜索分页与保留旧数据

## 本集目标

- 使用响应式 `queryKey`。
- 做搜索和分页。
- 理解 `keepPreviousData`。
- 理解 Abort 和竞态保护。

## 1. 响应式 queryKey

```js
const [page, setPage] = createSignal(1);
const [keyword, setKeyword] = createSignal('');

const products = createQuery({
  queryKey: () => ['products', { page: page(), keyword: keyword() }],
  keepPreviousData: true,
  queryFn: async ({ queryKey, signal }) => {
    const [, params] = queryKey;
    const url = `/api/products?page=${params.page}&keyword=${params.keyword}`;
    const response = await fetch(url, { signal });
    return response.json();
  },
});
```

口播：

`queryKey` 可以是函数。

函数里读到的 signal 会被追踪。

当 `page` 或 `keyword` 变化，query 会自动切换 key 并请求。

## 2. keepPreviousData

```js
keepPreviousData: true;
```

口播：

分页时，如果每次切页都清空列表，体验会闪。

`keepPreviousData` 会保留旧列表，同时把 `isStale` 设为 true。

UI 可以显示一个小的刷新状态，而不是整页 loading。

```js
createEffect(() => {
  renderList(products() || []);
  refreshBadge.hidden = !products.state.isFetching || products.state.isLoading;
});
```

## 3. reload 和 refetch

```js
products.refetch();
products.reload();
```

口播：

`refetch` 适合后台刷新，默认保留旧数据。

`reload` 适合用户明确要求重新加载，默认清空旧数据并进入 loading。

## 4. 高频搜索

```js
input.addEventListener('input', (event) => {
  setKeyword(event.currentTarget.value.trim());
  setPage(1);
});
```

口播：

搜索输入可能很快。

旧请求可能比新请求更晚返回。

`vanilla-query` 内部用 request id 做竞态保护，只接受最后一次请求的结果。

同时会把 `signal` 传给 `queryFn`，你用 fetch 时旧请求可以被 abort。

## 5. 手动 abort

```js
cancelButton.addEventListener('click', () => {
  products.abort();
});
```

口播：

abort 后，当前请求会被中断，旧结果不会写回 state。

这适合离开页面、关闭弹窗、取消搜索。

组件销毁时调用：

```js
products.destroy();
```
